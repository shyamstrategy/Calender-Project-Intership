## 💡 About The Project & My Choices

When approaching this challenge, I wanted to go beyond just rendering a grid of dates. I wanted it to feel organic with some UI magic sprinkled on top. Here's a brief look at some technical and design choices I made:

- **Next.js & React:** I used Next.js for its robust routing and fast performance. It makes managing state and structuring the project super clean.
- **Tailwind CSS:** For styling, Tailwind was a no-brainer. It allowed me to rapidly iterate on the visual aesthetics, ensuring the app is mobile-responsive and looks beautiful and modern on any device size.
- **`date-fns` vs Traditional Date Logic:** Date math can get tricky! I opted for `date-fns` because it's modular, lightweight, and gives me precise control over generating the monthly grid and handling multi-day range logic without bloating the JavaScript bundle.
- **Micro-interactions & Animations:** I implemented custom 3D page-flip animations and smooth month-to-month transitions to make navigating the calendar feel fun and engaging.
- **Lucide Icons:** Used `lucide-react` for clean, crisp, and consistent iconography throughout the UI.

### Core Features Included:
- Dynamic month grid generation
- Day-range selection for events (e.g., smoothly selecting a timeline from the 5th to the 8th)
- Fully mobile-responsive layout
- Month-specific themes, dynamic hero image accents, & holiday markers
- Integrated monthly notepad section for quick notes


   *(Be sure to replace the URL with the actual repo address)*

2. **Install dependencies:**
   Make sure you have NodeJS installed, then run:
   ```bash

   ```

3. **Start the development server:**
   ```bash
   npm run dev
   ```

4. **View the app:**
